# 访问控制运行时 QA 清单

本清单用于微信开发者工具和真机验证。不要把真实 OpenID、配对码、密码、AppSecret、云开发密钥、token、函数日志原文或环境变量真实值提交到公开仓库、截图说明或公开 issue。

## 1. 准备私有云开发配置

1. 在云开发控制台创建或更新 Event Function：
   - 函数名：`access-control`
   - 函数类型：Event Function
   - 运行时：`Nodejs18.15`，或在私有记录里写明实际运行时
   - 函数根目录：`cloudfunctions`
2. 只在云开发控制台或私有部署通道配置两人白名单：
   - owner 槽位：真实拥有者 OpenID
   - partner 槽位：真实伴侣 OpenID
   - 状态只使用 `active`、`disabled`、`revoked`
   - 不配置第三个允许访问的 OpenID
3. 私下应用真实数据库规则：
   - `love_entries` 只允许两人 OpenID 白名单读、建、改、删
   - 新增必须要求 `doc.coupleId == 'main'`
4. 私下应用真实云存储规则或等价私有权限：
   - 只允许两人 OpenID 白名单访问
   - 路径限制在 `love-entries/main/`
5. 如果启用配对码，仅在私有环境保存哈希和盐。本阶段访问页只显示“需要配对”状态，不输入配对码，也不调用 `verifyPairingCode`。

## 2. 微信开发者工具验证

1. 本地构建：
   ```bash
   pnpm build:mp-weixin
   ```
2. 用微信开发者工具导入：
   ```text
   dist/build/mp-weixin
   ```
3. 确认首次打开页面是：
   ```text
   pages/access/access
   ```
4. 打开调试器 Network / Console。不要复制真实日志到公开仓库。

## 3. 真机验证

1. 使用微信开发者工具预览码或体验版。
2. 分别用 owner 微信、partner 微信、未知微信扫码。
3. 每次切换账号后，重新进入小程序，避免沿用旧页面栈和旧运行时状态。
4. 只记录测试结论，不记录真实 OpenID、环境变量、配对码、函数日志原文或截图中的敏感内容。

## 4. 必测用例

### 4.1 owner 授权访问

1. 将 owner 槽位设为 `active`。
2. 用 owner 微信打开小程序。
3. 期望：
   - 先进入访问页。
   - 状态通过后跳转到首页。
   - 首页列表可正常读取。
   - 详情页、创建页、编辑页、上传图片、临时图片链接恢复均保持原有行为。

### 4.2 partner 授权访问

1. 将 partner 槽位设为 `active`。
2. 用 partner 微信打开小程序。
3. 期望与 owner 一致。

### 4.3 未知账号拒绝

1. 使用不在 owner / partner 槽位里的微信打开小程序。
2. 期望：
   - 停留在访问页。
   - 显示未放行状态。
   - 不进入首页。
   - Network / Console 中没有业务列表、详情、上传、临时链接请求先于访问校验发生。

### 4.4 撤销账号拒绝

1. 将某个已知槽位状态改为 `revoked`。
2. 用该微信打开小程序。
3. 期望：
   - 停留在访问页。
   - 显示已撤销或不能进入的安全提示。
   - 不加载业务数据。
4. 再将状态恢复为 `active`，重新进入小程序，确认可恢复授权访问。

### 4.5 停用账号拒绝

1. 将某个已知槽位状态改为 `disabled`。
2. 用该微信打开小程序。
3. 期望：
   - 停留在访问页。
   - 显示已停用或不能进入的安全提示。
   - 不加载业务数据。

### 4.6 云函数未部署

1. 在私有测试环境中临时移除、改名或不部署 `access-control` 函数。
2. 打开小程序。
3. 期望：
   - 访问页显示“访问校验云函数还没部署好，请稍后再试。”或同级安全错误。
   - 页面只显示重试，不暴露函数日志、OpenID、环境变量、栈信息或内部异常。
4. 恢复函数后重试，确认可以进入正常状态。

### 4.7 云函数已部署但未配置

1. 保留函数，但移除私有 owner / partner OpenID 配置，或只保留占位值。
2. 打开小程序。
3. 期望：
   - 访问页显示访问名单未配置类提示。
   - 不加载业务数据。
   - 不展示真实配置键值、哈希、盐、OpenID 或函数内部错误。

### 4.8 函数权限失败

1. 在私有测试环境中临时收紧函数调用权限，使小程序无法调用 `access-control`。
2. 打开小程序。
3. 期望：
   - 访问页显示“访问校验没有被云开发放行，请稍后再试。”或同级安全错误。
   - 不加载业务数据。
   - 不显示底层权限错误原文。
4. 恢复函数权限后重试。

### 4.9 网络超时或离线

1. 在微信开发者工具或真机中切换弱网、断网，或使用可控网络代理模拟超时。
2. 打开小程序并点击“再试一次”。
3. 期望：
   - 访问页显示网络较慢或访问校验未完成类提示。
   - 不加载业务数据。
   - 网络恢复后重试可进入正常状态。

### 4.10 需要配对状态

1. 在私有环境开启配对码要求，并确保哈希与盐只存在于云开发私有配置。
2. 用白名单用户打开小程序。
3. 期望：
   - 访问页只显示“还差一段小暗号”类状态。
   - 本阶段不出现输入框。
   - 页面不调用 `verifyPairingCode`。
   - 不加载业务数据。

### 4.11 业务页直达拦截

1. 在微信开发者工具编译模式中分别指定业务页：
   - `pages/index/index`
   - `pages/detail/detail`
   - `pages/create/create`
   - `pages/songs/songs`
   - `pages/song-edit/song-edit`
   - `pages/tasks/tasks`
   - `pages/task-edit/task-edit`
   - `pages/memos/memos`
   - `pages/memo-edit/memo-edit`
   - `pages/settings/settings`
   - `pages/design-preview/design-preview`
2. 使用未知账号或撤销账号打开。
3. 期望：
   - 页面调用访问校验后重定向到访问页。
   - 在授权确认前，不执行私有列表、详情、创建、更新、删除、上传、临时链接请求。

### 4.12 授权后业务行为不变

1. 使用 owner 或 partner 授权进入。
2. 逐项测试：
   - 首页列表读取。
   - 回忆详情读取。
   - 新建回忆。
   - 编辑回忆。
   - 删除回忆。
   - 上传图片。
   - 图片临时链接恢复。
   - 小歌单、小约定、小档案读取和修改。
3. 期望：
   - 授权后业务行为与 Phase 3 前一致。
   - collection 仍为 `love_entries`。
   - 记录仍写入 `coupleId: "main"`。
   - 存储路径仍为 `love-entries/main/`。

## 5. 每次运行前后检查

1. 本地检查：
   ```bash
   pnpm scan:security-baseline
   pnpm scan:access-control
   ```
2. 确认公开仓库只包含占位符、环境变量名和安全说明。
3. 确认没有把真实 OpenID、配对码、密码、AppSecret、云开发密钥、token、哈希、盐或函数日志原文写入任何文件。
