# AI 功能开发与视觉文案规范

本规范是后续 AI / Codex 在本项目中新增功能、修改页面、调整组件或补丁修复时必须默认遵守的长期指引。用户后续提出新需求时，即使没有再次说明“俏皮可爱、手绘风、小纸条感”，也必须自动按本规范执行。

## 1. 适用范围

本规范适用于所有未来改动，包括但不限于：

- 新页面
- 新组件
- 新卡片、容器、列表、空态、错误态、加载态
- 新按钮、入口、操作菜单、弹层、确认框
- 新表单、编辑页、详情页
- 新数据读取、缓存、刷新、保存、删除流程
- 现有页面的小范围视觉或文案修复

只要改动会影响用户看到或触摸到的界面，就必须执行本规范。

## 2. 当前产品身份

- 用户可见产品名：`小珊的树洞`
- 产品气质：只给两个人悄悄使用的小树洞、小本本、小纸条、小约定。
- 当前阶段：Stage 1 owner-only 单用户烟测。
- 不得暗示伴侣账号已接入、双方在线、公开社交、评论、分享、绑定、邀请或登录体系。

允许在用户可见文案中使用 `小珊的树洞` 作为产品名。仍然禁止出现完整真实姓名与无关个人信息。

## 3. 一句话总原则

每次新增功能或页面时，都要先问：

> 这像不像一张被悄悄放进小本本里的纸条？

如果实现看起来像后台管理、普通 CRUD、系统设置、企业待办、功能宫格、数据面板或商业 App 模板，就不符合本项目。

## 4. 视觉风格规则

必须保持：

- 调皮
- 可爱
- 俏皮
- 手绘风
- 小纸条感
- 小票根感
- 小印章感
- 私密树洞感
- 温柔长期记录感
- 两个人之间的小暗号感

可以使用以下视觉隐喻：

- 小纸条
- 小本本
- 小票根
- 小便签
- 小贴纸
- 小印章
- 手绘短线
- 轻微纸角
- 柔和红蓝线索
- 云朵便签
- 树洞 / 小角落 / 小仓库等轻隐喻

禁止做成：

- 后台管理系统
- 企业任务工具
- 普通表单提交页
- KPI / dashboard / 图表面板
- 商业品牌吉祥物风
- 高饱和儿童卡通
- 夸张表情包
- 玻璃拟物
- 复杂 3D
- 大量渐变
- 海洋、珊瑚礁、贝壳、珍珠、鱼、水母、海浪等海洋主题

## 5. 卡片、容器、列表设计规则

新增卡片或容器时，必须符合以下方向：

- 像纸条、票根、便签、小纸样或小本本里的夹页。
- 使用现有 `--app-*` token、`src/styles/mixins.scss` 和 app 组件。
- 不得直接写 raw hex、`rgba()`、固定 `px/rpx`、直接阴影、固定圆角、固定字号、固定动画时长。
- 不得为单个页面随意定义新的 `--app-*` token。
- 新容器不能只是“白色大卡片 + 普通标题 + 普通按钮”。必须有符合项目气质的层次、留白、轻装饰或纸感细节。

新增列表项时，优先考虑：

- 小票根列表
- 小纸签列表
- 小印章状态
- 手绘标签
- 轻量完成态
- 温柔空态

不要做：

- 表格
- KPI 卡片
- 通用后台列表
- 企业 todo 列表
- 大段系统说明面板

## 6. 文本文案规则

所有用户可见文案必须是简体中文，并符合项目语气。

推荐词汇：

- 轻轻
- 慢慢
- 悄悄
- 收好
- 放进来
- 藏起来
- 小纸条
- 小票根
- 小本本
- 小歌单
- 小约定
- 小回忆
- 小样本
- 小仓库
- 翻一翻
- 勾上
- 先放着

禁止过度卖萌：

- 宝贝
- 亲亲
- 么么
- 哭哭
- 呜呜
- 冲鸭
- 超级无敌
- 表情包式语气
- 幼儿化措辞

禁止用户可见英文 UI，例如：

- Save
- Cancel
- Delete
- Edit
- Task
- Todo
- Done
- Loading
- Error
- Upload
- Settings
- Preview
- Version
- Connected
- Comfortable
- Compact
- Dashboard
- Feature

技术变量名、类型名、依赖名和技术文档中的英文允许保留。

## 7. 高风险文案必须克制

以下场景可以温柔，但必须清楚、克制、明确：

- 删除确认
- 保存失败
- 上传失败
- 网络失败
- 权限失败
- 云开发失败
- 数据不可恢复提示
- owner-only 限制提示

推荐：

- `删除这条回忆`
- `这会删除这条记录和对应照片。`
- `取消`
- `删除`
- `这件小事暂时没删掉，请稍后再试。`
- `可能是网络慢了一点，稍后再试一次。`

禁止：

- `它会永远消失喔呜呜`
- `真的不要它了吗哭哭`
- `再也见不到它啦`
- 任何会降低删除/失败语义清晰度的卖萌文案

## 8. 页面类型规范

### 首页

首页必须像 `私密停靠处 / 小树洞入口`，不是 dashboard。

推荐语气：

- `把今天值得记住的小事收好`
- `夹在回忆本里的小纸签`
- `点一首，勾一件，慢慢收着。`

### 回忆

回忆相关页面像 `小日记纸页 / 被收好的小纸条`。

推荐语气：

- `把哪件小事收起来？`
- `想留下哪句话？`
- `私密相册夹`
- `轻轻收好`
- `翻这张小纸页`

### 点歌

点歌不是普通歌单管理，而是 `她想听我唱的歌先收着`。

推荐语气：

- `小歌单`
- `想听你唱的歌都放在这里`
- `先把歌名塞进来，唱不唱不着急。`
- `放进小歌单`

### 必做事项

事项不是企业 todo，而是 `以后想一起做的小事`。

推荐语气：

- `小约定`
- `想一起做的事慢慢勾`
- `不是压力清单，是给未来留的位置。`
- `轻轻勾上`
- `放回清单`

### 设置

设置不是系统设置页，而是 `小纸样本 / 悄悄设置`。

推荐语气：

- `小纸样本`
- `悄悄设置`
- `今天的小纸样`
- `白天夜里的样子`
- `小纸样颜色`
- `小纸条间距`
- `字要多大`

### 小宠物

小宠物是首页小导航员，不是聊天机器人，也不是伴侣在线状态。

推荐语气：

- `想去哪儿呀？`
- `我悄悄带你过去。`
- `写回忆`
- `小日子`
- `小歌单`
- `小约定`
- `换样子`
- `先藏起来`

## 9. 组件与控件规则

新增用户交互控件必须优先使用 Wot UI：

- `wd-button`
- `wd-input`
- `wd-textarea`
- `wd-picker`
- `wd-popup`
- `wd-message-box`
- `wd-toast`
- `wd-loading`
- `wd-cell`
- `wd-cell-group`
- 其他合适 Wot 组件

优先复用项目组件：

- `app-shell`
- `app-custom-nav`
- `empty-state`
- `entry-card`
- `image-grid`
- `app-option-group`
- `app-option-button`
- `theme-swatch-picker`

禁止在新业务页面或组件里使用原生业务交互控件作为主要 UI：

- 原生 `button`
- 原生 `input`
- 原生 `textarea`
- 原生 `checkbox`
- 原生 `radio`
- 原生 `switch`
- 原生 `picker`
- 原生 `slider`
- 原生 `form`

平台结构性 `view` / `text` / `image` 可以使用。

## 10. 数据加载体验规则

所有接口读取页面必须遵守当前本地缓存与静默刷新策略：

- 有缓存时先显示缓存，不要空白闪烁。
- 后台静默刷新。
- 没有真实业务变化时，不要重新替换列表对象。
- 只有 `tempFileURL` 变化时，不要让当前图片重新加载闪烁。
- 新建、编辑、删除、状态切换后必须同步缓存。
- 编辑页后台刷新不能覆盖用户正在输入的内容。

不得为了“刷新最新数据”牺牲页面稳定性。

## 11. 新需求执行流程

AI / Codex 每次实现新功能或新需求前，必须默认执行：

1. 阅读 `AGENTS.md`。
2. 阅读 `docs/AI_CODING_GUIDELINES.md`。
3. 阅读本文件。
4. 检查当前页面/组件已有风格。
5. 先确认最小安全改动范围。
6. 只做本轮目标。
7. 不重构底层架构。
8. 不绕过 repositories / services / composables。
9. 不新增产品边界之外的能力。
10. 不让新增 UI 看起来像普通模板。

## 12. 新功能自检清单

提交结果前必须自检：

- 新卡片是否像小纸条 / 小票根 / 小本本？
- 新容器是否符合当前页面节奏？
- 新按钮是否使用 Wot UI 或已批准 app 组件？
- 新文案是否俏皮可爱但不幼稚？
- 删除/错误/失败文案是否清楚克制？
- 是否无英文用户 UI？
- 是否无真实姓名？
- 是否无越界功能暗示？
- 是否无 raw style 违规？
- 是否未改 CloudBase 边界？
- 是否未破坏 owner-only？
- 是否通过项目验证命令？

## 13. 验证要求

页面、组件、样式、repository、缓存、文案或文档改动后，必须继续运行：

```bash
pnpm scan:ui-copy
pnpm scan:design-tokens
pnpm type-check
pnpm type-check:strict
pnpm build:mp-weixin
git diff --check
```

同时确认：

- 无 AppSecret / appsecret。
- 无 UnoCSS。
- 小宠物仍只在首页。
- CloudBase envId / AppID / collection / storage prefix 未改。
- 用户可见 UI 文案为简体中文。

## 14. AI 返回要求

实现结果必须说明：

- 是否遵守本规范。
- 新增或修改的卡片、容器、文本如何符合俏皮可爱手绘风。
- 是否有非文案/非视觉的业务改动。
- 是否完整运行验证。
- 仍需用户真机确认的事项。
